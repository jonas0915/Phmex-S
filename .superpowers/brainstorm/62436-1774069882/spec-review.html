Design Spec — Trading Desk Visual Overhaul

Review the full spec below. Click approve when ready to proceed to implementation planning.

Constraints

Dashboard isolation: Zero bot imports, zero API calls. Reads trading_state.json + bot.log only.
Performance: 30+ FPS on integrated GPU. Must not compete with the bot.
Structure preserved: Same room (12×10×4), desk positions, camera, HUD, social events, sleep system, lower floors.

1. Approach

Replace procedural geometry with GLTF model assets loaded via THREE.js GLTFLoader. Industry-standard for realistic browser 3D.

3. Event-Driven Animations (3 Tiers)

T1: Professional
Typing, monitor checks, phone calls
T2: Active Trading
Signals, entries, exits, pointing
T3: Wolf of Wall Street
High-fives, desk slams, celebrations
Key triggers: 3+ win streak → high-five | 5+ streak → team celebration | Big loss → desk slam | 3 losses → huddle | ATH → big celebration
Status indicators: Monitor glow (green/amber/red/blue) + desk LED strip + floating name/role label (replaces Sims plumbobs)
Social events preserved: Coffee (2m), facility visits (45s), 1-on-1s (30m), team meetings (1h), team events (5m), sleep (11pm-6am)

4. Environment — Salesforce Tower

Location: High floor of Salesforce Tower, SF. Views: Golden Gate (N), Bay Bridge (E), SoMa (S), downtown below (W).
HDRI: 4096×2048, used for PBR reflections only (scene.environment).
Panorama: Upgraded procedural on sky dome — SF landmarks, window grids, fog/haze, animated bay water.
Glass walls: MeshPhysical with transmission + IOR (perf fallback to opacity if needed).
Room: 12×10×4, stairwell, break room, conference room, lower floors — all unchanged.

5. Rendering Pipeline

Shadows: 1024→2048 | Pixel ratio: retina support | Tone mapping: ACESFilmic @ 1.0
Post-processing: RenderPass → SSAO (auto-toggle) → Bloom (0.08) → SMAA → Color Grading
Materials: Normal-mapped concrete floor, real glass, wood/leather grain desks, env-mapped monitor bezels, acoustic ceiling
Performance: LOD, SSAO half-res, single shadow light, animation mixer idle optimization, geometry instancing

6. Asset Pipeline

Total: ~15-20MB in assets/ directory (characters, animations, furniture, HDRI)
Loading: GLTFLoader, progress bar, browser-cached after first load
HTTP serving: New /assets/<path> route with path traversal protection, MIME types, 24hr cache
Fallback: Procedural capsule/sphere for failed character loads, colored boxes for furniture

7-8. HUD & Data Pipeline (Unchanged)

HUD: Bottom stats bar, right comms panel, left intel panel — same layout, same data, same rates.
Speech bubbles: Same system, dialogue trees rewritten for new agent names/roles.
Data: Reads trading_state.json + bot.log | /api/data polled 1s | Zero API calls | Port 8060
